**Критичность (Priority):** Medium (Средний)

**Серьезность (Severity):** Medium (Средняя)

**Шаги воспроизведения:**
1. Перейти на https://www.saucedemo.com/
2. Авторизоваться с логином "problem_user" и паролем "secret_sauce"
3. Просмотреть страницу с товарами (Inventory page)
4. Обратить внимание на изображения товаров

**Ожидаемый результат:** Каждый товар отображается с уникальным, соответствующим ему изображением (куртка, рюкзак, футболка и т.д.)

**Фактический результат:** Все товары отображаются с одинаковым изображением собаки с теннисным мячом во рту

## Тестовые сценарии и методы при поиске дефекта.

**Тестовые сценарии:**
1. Сравнительное тестирование функциональности для разных типов пользователей
2. Тестирование отображения контента и медиа-файлов
3. Проверка корректности загрузки и отображения изображений

**Методы тестирования:**
- **Сравнительное тестирование** - сравнение поведения системы для разных пользователей
- **Тестирование пользовательского интерфейса (UI Testing)** - проверка корректности отображения графического контента
- **Контент-тестирование** - проверка соответствия контента требованиям
- **Кросс-пользовательское тестирование** - проверка работы системы под разными учетными записями

## Данное поведение является дефектом по следующим причинам:

1. **Нарушение бизнес-логики** - в интернет-магазине изображения товаров являются критически важным элементом для принятия решения о покупке
2. **Несоответствие ожиданиям пользователя** - пользователь ожидает видеть реальное изображение товара, которое он покупает
3. **Снижение функциональности** - невозможно идентифицировать товар по изображению, что является базовой функцией любого e-commerce приложения

## Гипотеза о возможных причинах возникновения дефекта

**Технические причины:**
1. **Ошибка в обработке данных пользователя** - специальная логика для пользователя problem_user, которая неправильно формирует URL изображений
2. **Проблема с API или бэкендом** - бэкенд возвращает для данного пользователя некорректные пути к изображениям
3. **Ошибка в коде фронтенда** - компонент отображения товара содержит баг при работе с определенными условиями пользователя
4. **Проблема с роутингом изображений** - все запросы на изображения перенаправляются на один некорректный файл

**Точки исправления:**
1. **Бэкенд-код** - найти в коде сервера специальные условия для problem_user и убрать подмену изображений
2. **Фронтенд-компонент** - найти компонент, который отображает товары, и убрать условие, которое для problem_user подставляет неправильную картинку
3. **Логика маршрутизации** - найти конфигурацию сервера или правила перенаправления и убрать специальное поведение для изображений

## Влияние дефекта на пользователя и бизнес-процессы

**Риски и неудобства для пользователя:**
1. **Невозможность визуальной идентификации товара** - пользователь не может понять, что именно он покупает
2. **Дезориентация в каталоге** - сложно различить товары между собой
3. **Снижение доверия к магазину** - создает впечатление ненадежности или тестового характера платформы

**Бизнес-последствия:**
1. **Снижение конверсии в покупки** - пользователи не покупают товары, которые не могут рассмотреть
2. **Репутационные риски** - негативные отзывы о некорректной работе сайта
3. **Потеря продаж** - пользователи уходят к конкурентам с работающей системой отображения товаров
